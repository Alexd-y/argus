## 1. Бэклог (что сознательно не доведено)

### Из **v5** (`argus_implementation_prompt_v5.md` + отчёт `2026-04-03-argus-v5-orchestration-report.md`)

- **120+ новых MCP tools** под REST, которых в бэкенде ещё нет: pause/resume/retry scan, compare/clone/export scan, расширенный finding/report management, recon engagements, threat modeling, VA/exploitation по engagement, расширенный intelligence (CVE search, exploitdb, WAF/CDN, CT logs, whois, geoip и т.д.).
- Доведение до **150+ явных** `@mcp.tool()` (не только Kali registry) и полный **`test_mcp_tools.py`** по перечню v5 — после паритета API.
- **Блок 6 v5** (частично): timeline, statistics, false-positive POST, remediation GET и др. — если не покрыты отдельными задачами.
- **Усиление prod:** при пустом `ADMIN_API_KEY` cache-роутер может оказаться фактически открытым; согласовать политику (обязательный ключ или deny-by-default).

### Из **v4 / HexStrike-сравнения** (ранее в планах)

- Полный **TOOL_MAP** с парсерами вывода CLI, загрузка артефактов в MinIO для каждого инструмента.
- **~150 сырых CLI** как у HexStrike — не цель текущего ARGUS, но в бэклоге как продуктовый паритет.
- **ScanKnowledgeBase** в v5 уже сделан; дальше можно: глубже связать с отчётами, планировщиком сканов, UI.

### Из **документации**

- **Follow-up scan API** для Valhalla (`POST .../follow-up`) — в `docs/reporting.md` как будущая опция, сейчас ручной новый скан.
- **Аутентификация клиента к ARGUS API** (Bearer и т.д.) — в контракте фронта помечена как не реализованная.
- **Kubernetes / полноценный prod-hardening** — в архитектурных docs часто «out of scope, Compose».

### Операционное

- Полный **`pytest tests/`**, CI, canary — из отчётов Strix/v5 как рекомендации, не как «дыра в коде».

## 2. Неполные реализации и слабые места в коде

### Устаревшие **тексты** (поведение уже может быть нормальным)

- В **`mcp-server/argus_mcp.py`** у `get_scan_memory_summary`, `get_process_list`, `kill_process` в docstring может оставаться устаревший текст про «501 до реализации», хотя в **API роутерах `501` по grep не используется** — имеет смысл **обновить описания** под фактические ответы (200/403/404/JSON с ошибкой).

### Роутеры / сервисы

- **`api/routers/tools.py`** — при необходимости обновить шапку/комментарии под фактическую зрелость SaaS (маршруты → `execute_command`).
- **`core/auth.py`** — валидация API key: при необходимости уточнить комментарии про последующую проверку по БД.
- **`api/routers/auth.py`** — dev fallback идентификатор пользователя для части сценариев.
- **`orchestration/exploit_verify.py`** — безопасный PoC runner; отслеживание **ARGUS-008** при необходимости.
- **`api/schemas.py`** — в описании поля про **recovery** убрать устаревшие формулировки; recovery уже внедрён.

### Recon / адаптеры

- **`recon/adapters/security/*`** — минимальные парсеры до полного покрытия: `prowler_adapter`, `checkov_adapter`, `terrascan_adapter`, `scoutsuite_adapter`, `trufflehog_adapter`.
- **`data_sources/*`** — клиенты внешних API: HIBP, SecurityTrails, VirusTotal, Censys (проверить полноту `query()`).
- **`recon/va_http_audit.py`** — HTTP audit trail для артефактов (намеренно не полный реплей запросов).
- **`recon/step_registry.py`** — `STUB_STEPS` сейчас пустой; механизм шагов с audit-trail-only в пайплайне есть.

### Отчёты / шаблоны

- **`services/reporting.py`**, **`jinja_minimal_context.py`** — ключ **`tier_stubs`** / видимость секций по тиру: шаблонные подсказки по тиру в отчёте, не внешний API.

### MCP

- **`mcp-server/main.py`** — при необходимости обновить комментарий у entrypoint (stdio/HTTP).
