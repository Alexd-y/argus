Ниже — **карта веб-интерфейсов и доступов** для типичного локального запуска (Docker Compose из `infra/` + Next.js с хоста). **Конкретные пароли и ключи** берите только из **вашего** `infra/.env` и `Frontend/.env.local`; в репозитории они не должны совпадать с продом.

---

## 1. Основной веб-клиент (сканирование, отчёты)

| Параметр | Значение |
|----------|----------|
| **URL (dev)** | `http://127.0.0.1:5000` (скрипт `npm run dev` в `Frontend/`, порт задан в `Frontend/package.json`) |
| **Логин / пароль** | Отдельной формы «логин пользователя» для публичного скана обычно **нет**: запросы к API идут через rewrite на бэкенд (`Frontend/next.config.ts` → `NEXT_PUBLIC_BACKEND_URL`). Доступ к API с браузера обеспечивается **шлюзом nginx** (см. п. 4), а не логином в UI. |
| **Примечание** | В `Frontend/.env.local` задают `NEXT_PUBLIC_BACKEND_URL` (часто `http://127.0.0.1:8080` — через nginx). |

---

## 2. Админ-панель (тот же `Frontend`, App Router)

| Параметр | Значение |
|----------|----------|
| **URL** | `http://127.0.0.1:5000/admin` |
| **Вход** | `http://127.0.0.1:5000/admin/login` |
| **Логин / пароль** | Учётные записи **`admin_users`** в БД. Первичное создание — через bootstrap в конфиге бэкенда: переменные **`ADMIN_BOOTSTRAP_SUBJECT`**, **`ADMIN_BOOTSTRAP_PASSWORD_HASH`** (и роль/тенант), см. `backend/src/core/config.py` и операторский runbook `docs/operations/admin-sessions.md`. Пароль в `.env` обычно **не хранится открытым текстом**, а как **bcrypt-хэш**. |
| **Режимы** | `NEXT_PUBLIC_ADMIN_AUTH_MODE`: в режиме **`cookie`** (dev) логин может **редиректить сразу в `/admin`** без формы; для реальной сессии в проде ожидается **`session`**. |

---

## 3. Отдельное приложение `admin-frontend` (опционально)

| Параметр | Значение |
|----------|----------|
| **URL (dev)** | `http://127.0.0.1:3001` (`npm run dev` в `admin-frontend/`) |
| **В Docker Compose** | Отдельного сервиса «admin-frontend» в `infra/docker-compose.yml` **нет**; это **ручной** второй фронт, если вы его поднимаете. |
| **Доступ** | Зависит от того, как вы его подключите к API (те же принципы, что у основного фронта). |

---

## 4. Публичный API-шлюз (Nginx)

| Параметр | Значение |
|----------|----------|
| **HTTP** | `http://127.0.0.1:8080` (переменная **`ARGUS_HTTP_PORT`**, по умолчанию 8080) |
| **HTTPS** | `https://127.0.0.1:8443` ( **`ARGUS_HTTPS_PORT`**, по умолчанию 8443; нужна TLS-конфигурация в nginx) |
| **Логин / пароль** | Это **не** веб-форма: для проксируемых `/api/*` nginx может подставлять **`ARGUS_GATEWAY_API_KEY`** в заголовок **`X-API-Key`** (значение из `infra/.env`). Админские маршруты — заголовок **`X-Admin-Key`** = **`ADMIN_API_KEY`**. |
| **Health** | Например `GET http://127.0.0.1:8080/api/v1/health` (публичный smoke-check). |

---

## 5. Прямой бэкенд FastAPI (обычно не для браузера)

| Параметр | Значение |
|----------|----------|
| **URL** | В базовом `docker-compose` порт **8000 на хост не проброшен**; доступ изнутри сети: `http://backend:8000`. В dev с монтированием кода бэкенд слушает 8000 **внутри контейнера**. |
| **Доступ** | Те же **API-ключи** / сессии, что и у API за nginx; для локальной отладки иногда пробрасывают порт отдельно — это уже ваша кастомизация. |

---

## 6. Консоль MinIO (объектное хранилище)

| Параметр | Значение |
|----------|----------|
| **URL** | `http://127.0.0.1:9001` (при dev-override с пробросом портов 9000/9001) |
| **Логин** | **`MINIO_ACCESS_KEY`** из `infra/.env` |
| **Пароль** | **`MINIO_SECRET_KEY`** из `infra/.env` |

---

## 7. Маршрут MCP в основном фронте (опционально)

| Параметр | Значение |
|----------|----------|
| **URL** | `http://127.0.0.1:5000/mcp` при **`NEXT_PUBLIC_MCP_ENABLED=true`** (см. `Frontend/README.md`) |
| **Доступ** | Настраивается через SDK/сессию; отдельной пары логин/пароль в документации как «фиксированный UI-логин» не задано. |

---

## 8. MCP HTTP-сервер (не полноценный «сайт», а API)

| Параметр | Значение |
|----------|----------|
| **Хост/порт** | В `infra/.env.example`: **`MCP_HTTP_HOST`**, **`MCP_HTTP_PORT`** (часто `127.0.0.1:8765`) для HTTP-транспорта |
| **Доступ** | При необходимости — **`MCP_AUTH_TOKEN`** (статический bearer), плюс политики JWT/API-key по коду MCP. |

---

## 9. Облако / туннель (если включите)

| Параметр | Значение |
|----------|----------|
| **Cloudflare Tunnel** | Профиль **`tunnel`** в Compose, токен **`CLOUDFLARE_TUNNEL_TOKEN`** — внешний URL задаётся в Zero Trust, не в репозитории. |

---

### Краткий итог

- **Браузер для оператора:** `http://127.0.0.1:5000` (скан) и `http://127.0.0.1:5000/admin` (админка с логином **subject/password** из БД/bootstrap).  
- **Единая точка API снаружи стека:** `http://127.0.0.1:8080` — без «логина на странице», зато с **ключами в заголовках** (шлюз и админка).  
- **S3-консоль:** `http://127.0.0.1:9001` — **MINIO_ACCESS_KEY / MINIO_SECRET_KEY**.

Если нужно, могу выписать **ровно ваши** URL и имена переменных из текущих `infra/.env` и `Frontend/.env.local` **без вывода значений секретов** (только checklist «что проверить»).
