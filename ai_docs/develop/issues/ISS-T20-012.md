# ISS — Backdrop role / onClick consistency in `FindingsTable` drawer

**Issue ID:** ISS-T20-012
**Owner:** Frontend / Admin console + a11y
**Source task:** T20 (Cycle 6 Batch 3 — global findings triage console)
**Source review:** post-review fix-cycle for commit `a27c07bf`
**Status:** **DEFERRED** — a11y nit (S3-1)
**Priority:** LOW (focus management already covered by S2-1; this is
purely about the backdrop element's semantic role)
**Date filed:** 2026-04-21

---

## Context

The findings detail drawer in
[`Frontend/src/components/admin/findings/FindingsTable.tsx`](../../../Frontend/src/components/admin/findings/FindingsTable.tsx)
renders a backdrop overlay behind the dialog:

```tsx
<div
  className="fixed inset-0 bg-black/40"
  onClick={onClose}
/>
```

The S2-1 fix (focus trap + auto-focus + restore-focus) made the
drawer keyboard-accessible. The backdrop, however:

* Has no `role` attribute (so screen readers don't announce it as a
  dismissible region).
* Has no `aria-label` (so the click target is invisible to
  AT users).
* Uses `onClick` without a matching keyboard handler (mouse-only
  dismiss path).

The dialog itself is keyboard-dismissible via `Escape` (S2-1), so
the backdrop is **redundant** for AT users — but it is still rendered
and may confuse axe-core (T26) once it lands.

## Problem

* axe-core (T26) is likely to flag the click-only backdrop as
  "interactive element with no keyboard equivalent" once the gate
  is wired.
* No semantic hint that clicking outside the drawer dismisses it
  (mouse users discover this only by trial).

## Proposal

Two acceptable patterns; pick one:

### Pattern A — make the backdrop a button (preferred)

```tsx
<button
  type="button"
  aria-label="Закрыть детали находки"
  className="fixed inset-0 bg-black/40 cursor-default"
  onClick={onClose}
/>
```

* Native button → keyboard-activatable via Enter/Space.
* `aria-label` declares its purpose.
* `cursor-default` keeps the visual identical to a plain backdrop.

### Pattern B — keep div + add `role="presentation"`

```tsx
<div
  role="presentation"
  className="fixed inset-0 bg-black/40"
  onClick={onClose}
  // explicitly tell AT this is decorative; Escape handles keyboard dismiss
/>
```

* Acknowledges the backdrop is a mouse-only convenience.
* Keyboard users dismiss via the existing Escape handler.
* Avoids axe-core warning (decorative role).

## Acceptance criteria

* (a) axe-core scan of `/admin/findings` with the drawer open returns
  0 critical / serious violations attributable to the backdrop.
* (b) Existing focus-trap / Escape behaviour unchanged (regression
  test in `FindingsTable.test.tsx` still passes).
* (c) Visual smoke: clicking outside the drawer still dismisses it.
* (d) Keyboard smoke: Tab/Shift+Tab cycles within the drawer (does
  not jump to the backdrop).

## Why deferred from T20 fix cycle

* S3-1 grade: a11y nit; the existing Escape + focus-trap is
  sufficient for keyboard users.
* Best paired with T26 (axe-core CI gate) so the regression test
  has a CI signal to lean on.

## Dependencies

* T26 (axe-core CI gate) — provides the automated detection.

## Rationale

Either pattern resolves the lint flag and clarifies the backdrop's
intent. Pattern A is preferred because it preserves the
"click-anywhere-outside to dismiss" UX for keyboard users too.
