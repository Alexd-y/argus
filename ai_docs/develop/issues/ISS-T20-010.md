## Context

[`Frontend/src/components/admin/findings/FindingsTable.tsx`](../../../Frontend/src/components/admin/findings/FindingsTable.tsx)
duplicates the `gridTemplateColumns` track string between the header
row and each data row:

```tsx
// header row
<div style={{ gridTemplateColumns: "minmax(0, 1.4fr) ... " }}>...</div>

// data row (inside the virtualizer)
<div style={{ gridTemplateColumns: "minmax(0, 1.4fr) ... " }}>...</div>
```

Both strings must stay in sync; otherwise header columns desynchronise
from data columns on resize. There is no compile-time guarantee they
match.

## Problem

* Magic string repeated in two places — drift risk on every column
  change.
* Implicit dependency on whitespace-identical strings.
* Linter cannot catch a one-character typo.

## Proposal

1. **Extract** the column track into a module-level constant:
   ```tsx
   const FINDINGS_GRID_COLUMNS =
     "minmax(0, 1.4fr) " +              // title
     "120px 100px 100px 100px " +        // severity / KEV / SSVC / EPSS
     "minmax(0, 1fr) " +                 // tenant
     "180px 80px";                       // updated_at / actions
   ```
2. **Reference** from both header + data rows:
   ```tsx
   <div style={{ gridTemplateColumns: FINDINGS_GRID_COLUMNS }}>...</div>
   ```
3. **Optional** — promote to CSS custom property
   `--findings-grid-cols` on a parent and consume via Tailwind
   `style={{ gridTemplateColumns: "var(--findings-grid-cols)" }}`,
   so the test for "header columns align with data columns" is just
   "they share a CSS var".
4. **Test** — Vitest assertion that `getComputedStyle(headerRow)
   .gridTemplateColumns === getComputedStyle(dataRow).gridTemplateColumns`
   on a mounted instance.

## Acceptance criteria

* (a) Single source of truth for the column track
  (constant or CSS var).
* (b) Vitest assertion proves header + data row stay in sync.
* (c) No visual regression (Playwright snapshot if available;
  otherwise manual verification noted in PR).
* (d) Existing 18 Vitest cases pass.

## Why deferred from T20 fix cycle

* S2-6 grade: pure code-organisation nit.
* Reviewer flagged it for visibility but not as a blocker; the fix

## Dependencies

* None. Pure component-internal refactor.

## Rationale

Eliminates a class of "header / data column drift" bugs that are
trivial to introduce and visually obvious only at certain viewport
widths.
