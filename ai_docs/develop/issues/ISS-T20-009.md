## Resolution

This issue was opened by the post-review reviewer to flag inconsistent
header naming between the original T20 implementation and the
backend's canonical names. The fix-cycle commit
(`fix(admin-ui): server action + correct filters + a11y for findings
(T20)`) standardised on the backend names everywhere:

| Old (frontend) | New (canonical, backend) | Surface |
|---|---|---|
| `X-Operator-Role` | `X-Admin-Role` | `Frontend/src/lib/serverAdminBackend.ts`, `Frontend/src/app/admin/findings/actions.ts` |
| `X-Tenant-ID` | `X-Admin-Tenant` | same as above |
| (missing) | `X-Operator-Subject` | `Frontend/src/app/admin/findings/actions.ts` (S1-3 fix) |

Verification:

* `grep -rn "X-Operator-Role" Frontend/src/` → no matches.
* `grep -rn "X-Tenant-ID" Frontend/src/` → no matches.
* All admin server actions now resolve `(role, tenant, subject)` from
  [`Frontend/src/services/admin/serverSession.ts`](../../../Frontend/src/services/admin/serverSession.ts)
  and forward them under the canonical `X-Admin-*` / `X-Operator-Subject`
  names that match `backend/src/api/routers/admin.py:require_admin`
  + `backend/src/api/routers/admin_bulk_ops.py:_operator_subject_dep`.
