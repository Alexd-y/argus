## Phase 1 — Implementation Complete (Cycle 6 Batch 6)

### What was built

**Option B selected: Backend session table** (recommended by user for speed + minimal complexity)

#### Backend (B6-T08)

- **Alembic 028:** `admin_sessions` table (cross-tenant, RLS DISABLED):
  - `id: uuid` (PK)
  - `email: varchar(320)` (operator email; indexed)
  - `session_token_hash: varchar(64)` (SHA256 hex of `HMAC(raw_token, ARGUS_ADMIN_SESSION_PEPPER)`; unique indexed)
  - `created_at: timestamp` (UTC)
  - `last_seen_at: timestamp` (updated on `/auth/admin/whoami` for observability)
  - `expires_at: timestamp` (session TTL: default 14 days from `created_at`)
  - `revoked_at: timestamp nullable` (admin logout or revocation)
  - `ip: inet nullable` (optional client IP logging)
  - `user_agent: text nullable` (optional client UA logging)

- **DAO:** `backend/src/auth/admin_sessions.py`
  - `create(email: str, ttl_seconds: int) → (raw_token: str, session_id: uuid)` — generates CSPRNG token + stores hash.
  - `lookup_by_token_hash(token_hash: str) → AdminSession | None` — verifies expiry + revocation.
  - `rotate(session_id: uuid, new_ttl_seconds: int) → (new_raw_token, new_session_id)` — proactive rotation (optional, for future idle-timeout).
  - `revoke(session_id: uuid)` — marks `revoked_at`.
  - `list_active(email: str | None) → list[AdminSession]` — for audit + session management UI.
  - `prune_expired()` — Celery beat task (hourly) deletes expired + revoked rows after 30-day soft-delete grace.

- **Verification:** `backend/src/auth/admin_users.py`
  - `verify_bootstrap_password(email: str, password: str) → bool` — bcrypt verification against `ARGUS_ADMIN_BOOTSTRAP_PASSWORD_BCRYPT` env.
  - Bootstrap admin loaded at startup if env set (one-time init).

- **Endpoints:** `backend/src/api/routers/admin_auth.py`
  - `POST /auth/admin/login {email: str, password: str}` → 200 + HttpOnly Secure SameSite=Strict cookie:
    - Bcrypt verification via `verify_bootstrap_password`.
    - Rate-limited: 5 attempts per 15 minutes per IP (via Redis or in-memory cache).
    - Response: `Set-Cookie: argus.admin.session=<raw_token>; HttpOnly; Secure; SameSite=Strict; Path=/; Max-Age=1209600`.
    - Audit emit: `admin.login` (email, ip, user_agent if available).
  - `POST /auth/admin/logout` → 200 + clears cookie:
    - Revokes session row (sets `revoked_at`).
    - Audit emit: `admin.logout`.
  - `GET /auth/admin/whoami` → 200 `{email: str, role: str, authenticated_at: datetime}` or 401:
    - Returns current authenticated admin's identity (server ↔ server, or middleware check).
    - Updates `last_seen_at` for observability.

- **Dual-mode auth dispatch:** `backend/src/api/dependencies.py::require_admin`
  - Reads `ADMIN_AUTH_MODE` env (`cookie` | `session`).
  - `cookie` mode (default, backward-compat):
    - Trusts `X-Admin-Role`, `X-Admin-Tenant`, `X-Operator-Subject` headers (set by Frontend from cookies).
    - Existing path; no changes.
  - `session` mode (new):
    - Parses `argus.admin.session` cookie (raw token).
    - Calls `admin_sessions.lookup_by_token_hash(hash(token + pepper))`.
    - On hit: constructs `ServerAdminSession` from row; verifies expiry + revocation.
    - On miss/expired/revoked: 401 Unauthorized.
    - Forwards `X-Operator-Subject=email` to audit pipeline.

- **Configuration:** `backend/src/core/config.py` new Settings:
  - `ADMIN_AUTH_MODE: Literal['cookie', 'session'] = 'cookie'`
  - `ARGUS_ADMIN_BOOTSTRAP_EMAIL: str` (required if bootstrap enabled)
  - `ARGUS_ADMIN_BOOTSTRAP_PASSWORD_BCRYPT: str` (bcrypt hash; can be generated offline)
  - `ARGUS_ADMIN_SESSION_PEPPER: str` (random string, 32+ bytes; mixes with token before hashing)
  - `ADMIN_SESSION_TTL_SECONDS: int = 1209600` (14 days)
  - `ADMIN_SESSION_SOFT_DELETE_DAYS: int = 30` (beat prune keeps rows for 30d post-revocation)

- **Production guard:** `backend/src/core/settings.py` `model_validator`:
  - If `production=True` AND `ADMIN_AUTH_MODE='cookie'` → raise `ConfigurationError` (prevents accidental dev shim in prod).

#### Frontend (B6-T09)

- **Dual-mode resolver:** `Frontend/src/services/admin/serverSession.ts`
  - Checks `process.env.NEXT_PUBLIC_ADMIN_AUTH_MODE` (defaults to `'cookie'` if not set).
  - `'cookie'` mode: existing path (reads from cookies; byte-equivalent to old code).
  - `'session'` mode: calls `callAdminBackendJson('/auth/admin/whoami', {method: 'GET'})` server-to-server; maps to `ServerAdminSession`.
  - Fallback: if `NEXT_PUBLIC_ADMIN_DEV_MODE='true'`, uses legacy `NEXT_PUBLIC_ADMIN_DEV_*` env vars (for local dev).

- **Login client:** `Frontend/src/services/admin/clientLogin.ts`
  - POST `/auth/admin/login {email, password}` proxy; error handling + redirect.

- **Login page:** `Frontend/src/app/admin/login/page.tsx` (new)
  - Server Component wrapper + `LoginClient.tsx` (Client Component).
  - Form: email + password input; submit via server action.
  - On success: 302 redirect `/admin` (or `?from=` query param).
  - On 401/rate-limit: show error.

- **Logout button:** `Frontend/src/app/admin/AdminLayoutClient.tsx` (updated)
  - New `LogoutButton` component (visible only if `NEXT_PUBLIC_ADMIN_AUTH_MODE='session'`).
  - POST `/auth/admin/logout`; redirects to login page.

- **Middleware:** `Frontend/src/middleware.ts` (new or extended)
  - If `NEXT_PUBLIC_ADMIN_AUTH_MODE='session'` AND request path matches `/admin/**`:
    - Check for `argus.admin.session` cookie.
    - If missing: 302 `/admin/login?from={original_path}`.
    - Otherwise: allow through.

- **Environment:** `Frontend/.env.example`
  - `NEXT_PUBLIC_ADMIN_AUTH_MODE=cookie` (default for dev).
  - Production `.env.production` sets `NEXT_PUBLIC_ADMIN_AUTH_MODE=session`.

- **Production guard (instrumentation):** `Frontend/instrumentation.ts` (new or extended)
  - If `NEXT_PUBLIC_PRODUCTION='true'` AND `NEXT_PUBLIC_ADMIN_AUTH_MODE='cookie'` → log warning at init (non-blocking; BE hard gate is authoritative).

#### Testing

- **Backend tests:** `backend/tests/auth/test_admin_auth_endpoints.py`
  - Happy path: login → 200 + cookie + audit log.
  - Rate-limit path: 6th attempt within 15min → 429.
  - Invalid password: 401.
  - Logout: 200 + session revoked + cookie cleared.
  - Whoami: 200 + correct email.
  - Whoami after logout: 401.

- **Frontend tests:** `Frontend/tests/e2e/admin-session-auth.spec.ts` (Playwright project, separate from legacy cookie-mode)
  - Happy path: login page loads → enter creds → redirect `/admin`.
  - Invalid password: error shown.
  - Logout button visible + clickable → redirects to login.
  - Middleware guards: direct access to `/admin` without cookie → 302 login.

### Verification

- (a) ✅ **Unique subject per session:** `admin_sessions.email` stored per row; session_id UUID-based; audit rows emit `X-Operator-Subject={email}`.
- (b) ✅ **Audit rows operator-unique:** Query `audit_logs WHERE actor='admin.login' ORDER BY created_at` shows distinct email values per session.
- (c) ✅ **Cookie tampering ineffective:** Browser dev-tools can modify `argus.admin.session` cookie value; backend validates token_hash → hash mismatch → 401. Cannot forge valid hash without knowing pepper.
